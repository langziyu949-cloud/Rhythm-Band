## 第一关：已确认的玩法设计

**主题**：小狗在快递站给包裹盖章。游戏过程中不出现“PERFECT / MISS”等文字，只用包裹、盖章和小狗的动作表达结果。

**节奏**：120 BPM，30 秒，共 15 个四拍小节。

| 段落 | 节奏型 | 画面与操作 |
| --- | --- | --- |
| 前 4 小节 | 倒计时 + 自动示范 | 小包裹来到传送带，小狗自动完成一次盖章。 |
| A 型普通包裹 | `｜x-x-o---｜` | 两个提示音后，玩家在第 3 拍点按一次；盖 1 个章。 |
| B 型大包裹 | `｜xxx-o-o-｜` | 三个提示音后，玩家连续点按两次；盖 2 个章。 |
| 结尾 | A/B 组合 | 连续混编后结束并结算。 |

**关卡编排**：`intro → demo → A → A → B → A → B → B → A → B → A → A → B → B → B`。

**反馈原则**：

- 准确：小狗迅速下压盖章，包裹与印章闪亮、弹起。
- 失误：小狗手忙脚乱晃动，包裹摇晃，并出现一个图形化的红色叉印。
- 不在关卡过程中显示文字评价；结算页可以保留文字成绩。

## 视觉与体验约束

- 风格参考《节奏天国》：扁平、轮廓清晰、色彩大胆、表情与动作夸张。
- 现在的第一关素材是临时可用版本；整体美术风格仍可继续探索，先保持“简单、扁平、读秒清楚”。
- 关卡进行中优先用角色动作、物体状态、粒子和颜色反馈信息，不依赖说明文字。
- 面向手机触屏；要保持竖屏、小屏易读，并避免需要精确小按钮的操作。

## 技术要点

- 项目是无构建步骤的静态 HTML/CSS/JS；修改 `index.html` 后即可在浏览器打开测试。
- `LEVELS` 数组定义五关数据；第一关的专属逻辑在 `Game` 中以 `stampMode` 分支实现。
- 第一关时间轴使用 `performance.now()` 与预渲染 BGM 对齐；其他原型关用 Web Audio 即时合成节拍。
- 重新生成第一关 BGM：`node tools/generate_stamp_station_bgm.js`。
- 第 1 关 BGM 的设计假设为 **120 BPM × 60 拍 = 30 秒**，改节奏表时要同时检查音乐、`LEVELS[0]` 和包裹动画提示。
